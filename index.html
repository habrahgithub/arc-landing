The 4-Tier Decision Engine

ALLOW

Low-risk changes like standard UI components or unit tests. Auto-validated and save proceeds normally.

WARN

Medium-risk edits like minor config changes. Requires explicit operator acknowledgment before proceeding.

REQUIRE_PLAN

High-risk logic modifications. Forces the operator to link a structured, pre-approved Blueprint Proof artifact.

BLOCK

Critical policy violations or unauthorized secure zone modifications. Save event is strictly blocked.
